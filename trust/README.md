## Gates

Fast syntactic gate:

```bash
trust/scripts/check-all.sh
```

Semantic gate, after `lake build`:

```bash
trust/scripts/check-all-semantic.sh
```

Full refresh after a legitimate trust-surface change:

```bash
trust/scripts/regenerate.sh
```

`check-all.sh` runs the V1 checks: locality, baseline freshness, forbidden
OUTPUT-EQ parameter shapes, sabotage floors and cross-witnessing, zero `sorry`,
uniform opcode coverage, canonical caller-burden drift, wrapper caller-burden
drift, ArithTable opcode-axiom retirement, and the shrinkage floor in
[`trust/.shrinkage-floor`](.shrinkage-floor).

`check-all-semantic.sh` runs the V2 checks over built oleans: per-theorem axiom
dependency drift, forbidden binder types after reducible unfolding, and exact
global-closure equality against `baseline-zisk-riscv-compliant.txt`.

## Changing Trust

To add trust, use the narrow path:

1. Put the declaration only in a file listed by
   [`allowed-axiom-files.txt`](allowed-axiom-files.txt).
2. Document its class and rationale in [`trusted-base.md`](trusted-base.md).
3. Regenerate the baselines with `trust/scripts/regenerate.sh`.
4. Commit the Lean declaration, baseline diffs, and documentation together.

To remove trust:

1. Delete the declaration and all now-unused proof plumbing.
2. Regenerate baselines with `trust/scripts/regenerate.sh`.
3. Update [`trusted-base.md`](trusted-base.md) and, if needed,
   [`trust/.shrinkage-floor`](.shrinkage-floor).
4. Review the ledger diffs as the audit surface.

## Anti-Laundering Terms

Use these terms consistently in plans, PRs, and review:

| Term | Meaning |
| --- | --- |
| Promise hypothesis | A caller-supplied assumption on an equivalence theorem that still needs derivation from trusted inputs or proved code. |
| Promise discharge | Replacing a promise hypothesis with a derivation from existing trust-ledger items and proved Lean code. |
| Discharge bridge | A proved lemma that connects a validator, row constraint, or trusted input to the promise being removed. |
| Trust ledger | The explicit Lean trust declarations and their documentation in this directory. |
| Caller-burden ledger | The generated binder inventory for canonical theorems and wrappers. |
| Anti-laundering metric | The hypothesis-count and caller-burden diffs that show whether a refactor reduced trust rather than renamed or split it. |
| Constructibility | The requirement that strengthened validators correspond to real ZisK trace constraints, not impossible assumptions. |

A promise-discharge PR is real progress only when the anti-laundering metric
shrinks, or when a documented structural-unpacking exception explains why a
temporary local increase collapses at the global theorem.
