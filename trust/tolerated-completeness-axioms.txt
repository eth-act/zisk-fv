# Tolerated non-closure axioms (V2 gate allowlist)

# Most entries here are completeness-direction axioms. They live in
# `ZiskFv/AirsClean/Completeness.lean` as the `circuit.completeness`
# field of each Clean `GeneralFormalCircuit`. Clean requires both a
# soundness and a completeness proof for every component; zisk-fv is a
# soundness-only verification project, so completeness for these circuits is
# declared as documented trust rather than hidden behind prover hypotheses.
#
# The V2 `check-closure-vs-baseline` gate treats entries in this file as
# source trust declarations whose absence from the global theorem's
# transitive axiom closure is expected, rather than as accidental dead
# trust. Every non-completeness entry must be documented in
# `trust/trusted-base.md`.
#
# Adding new entries to this allowlist requires CODEOWNER review (the
# `/trust/` path is in `.github/CODEOWNERS`).
#
