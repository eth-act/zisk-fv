# Nix build migration — implementation plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the Docker-based build pipeline (`docker/Dockerfile.{pilout,sail-lean}` + wrapper scripts) with a single `flake.nix` that produces `build/sail-lean/`, `build/zisk.pilout`, and `ZiskFv/Extraction/*.lean`. Eliminate the opam-style transitive-dep drift exposed by CI run 25192847660.

**Architecture:** Single flake at the repo root. Inputs include nixpkgs (NixOS 25.05), and source tarballs for Sail, sail-riscv, ZisK, pil2-compiler, pil2-proofman pinned by commit. Derivations: `pil-extract` (Rust binary), `sail-lean-tree` (overrides nixpkgs `sail-riscv` to enable the Lean target), `zisk-pilout` (cargo + npm + Node compose), `extracted-lean` (composes the above two). `apps.populate` copies derivation outputs into the repo paths `lake build` expects. `lake build` itself runs outside Nix in the devshell, using elan + mathlib's azure cache.

**Tech Stack:** Nix flakes, nixpkgs (`sail`, `sail-riscv`, `rustPlatform.buildRustPackage`, `buildNpmPackage`, `nodejs_20`), GitHub Actions, Lake/Lean (unchanged).

**Phasing:** Tasks 1–7 produce a working flake replacing the Docker pipeline. Tasks 8–10 wire it into CI and `bin/test.sh`. Tasks 11–12 delete docker/ and refresh docs. Each phase is independently testable. Realistic expectation: tasks 4 and 5 (`sail-lean-tree`, `zisk-pilout`) require iteration on derivation specifics; budget multiple build cycles per task.

    mkdir -p $out
    cp -r . $out/
  '';

Run `nix build .#pil2-compiler -L` once; it'll fail with the correct `npmDepsHash`. Paste it in, rebuild.

- [ ] **Step 2: Write the zisk-pilout derivation**

```nix
# nix/zisk-pilout.nix
{ stdenv, lib, rustPlatform, nodejs_20, makeWrapper,
  zisk-src, pil2-compiler, pil2-proofman-src,
  # System libs that zisk's cargo workspace's transitive C++ deps want:
  pkg-config, gmp, openssl, libsodium, libsecp256k1, nlohmann_json,
  protobuf, libuuid, grpc, libpqxx, nasm, openmpi, libclang, clang }:

let
  rust = rustPlatform.rust;
in stdenv.mkDerivation {
  pname = "zisk-pilout";
  version = "git-${builtins.substring 0 7 zisk-src.rev}";

  srcs = [ zisk-src ];
  sourceRoot = zisk-src.outPath;

  nativeBuildInputs = [
    rust.cargo rust.rustc nodejs_20 pkg-config makeWrapper clang
  ];
  buildInputs = [
    gmp openssl libsodium libsecp256k1 nlohmann_json protobuf
    libuuid grpc libpqxx nasm openmpi libclang
  ];

  # Vendored Cargo deps via Cargo.lock from zisk-src
  cargoDeps = rustPlatform.importCargoLock {
    lockFile = "${zisk-src}/Cargo.lock";
  };

  postUnpack = ''
    chmod -R +w $sourceRoot
  '';

  configurePhase = ''
    runHook preConfigure
    # Cargo expects the vendored registry layout
    export CARGO_HOME=$NIX_BUILD_TOP/cargo-home
    mkdir -p $CARGO_HOME
    ln -s ${cargoDeps} $CARGO_HOME/vendor
    cat > $CARGO_HOME/config.toml <<EOF
    [source.crates-io]
    replace-with = "vendored-sources"
    [source.vendored-sources]
    directory = "$CARGO_HOME/vendor"
    EOF

    # lib-float stub (mirrors the docker workaround)
    mkdir -p lib-float/c/lib
    touch lib-float/c/lib/libziskfloat.a lib-float/c/lib/ziskfloat.elf
    runHook postConfigure
  '';

  buildPhase = ''
    runHook preBuild

    # Build + run the 3 fixed_gen Cargo binaries to populate /tmp/fixed
    mkdir -p tmp/fixed
    for bin in arith_frops_fixed_gen \
               binary_basic_frops_fixed_gen \
               binary_extension_frops_fixed_gen; do
      cargo run --release --offline --bin $bin
    done

    # Compile PIL via Node + pil2-compiler
    node --max-old-space-size=12288 \
      ${pil2-compiler}/src/pil.js pil/zisk.pil \
      -I pil,${pil2-proofman-src}/pil2-components/lib/std/pil,state-machines,precompiles \
      -o $TMPDIR/zisk.pilout -u tmp/fixed -O fixed-to-file

    runHook postBuild
  '';

  installPhase = ''
    cp $TMPDIR/zisk.pilout $out
  '';
}
```

- [ ] **Step 4: Build (expect iteration)**

Run: `nix build .#zisk-pilout -L`
Expected on first try: fails on `cargoDeps` hash, missing C++ system dep, or `cargo run --offline` complaining about a network-needed dep. Iterate. Common gotchas:
- `pil2-proofman`'s starks-lib-c crate has a build.rs invoking `cmake` on a vendored C++ project — adds GMP / sodium / GRPC requirements you'll see in errors first.
- Node's `require` inside pil2-compiler may want a writable `node_modules` — symlink rather than copy, or pre-populate.
- The `--max-old-space-size=12288` (12GB) was set lower than upstream's 16GB to avoid OOMs on smaller runners; locally you can keep 12GB or raise.

Keep iterating until `result` is a 5–7 MB binary file (the .pilout).
